# Widget ↔ Engine JSON Contract

The **single canonical shape** flowing between the frontend Vue widgets and the
Python engine for `WP_Context` (and any future Pipeline Context node) is the
`ContextWidgetValue` JSON object below. Both sides MUST round-trip cleanly:

- Frontend: `parseWidgetJson` / `serializeWidgetJson` in
  `src/widgets/_shared.ts`
- Backend: `module_from_dict` / `module_to_dict` in `engine/modules.py`

If you change the shape on one side, change it on the other in the same PR
**and** add round-trip tests covering the new field.

---

## `ContextWidgetValue`

```jsonc
{
  "version": 1,
  "modules": [ /* ModuleEntry[] */ ]
}
```

| Field | Type | Notes |
|---|---|---|
| `version` | `1` (literal) | Bump on breaking shape changes; readers should fall back to empty when unknown. |
| `modules` | `ModuleEntry[]` | Ordered list. Order is significant — runtime applies modules top→bottom (last write wins per variable). |

### Module ordering semantics

Order in the `modules` array IS the runtime application order. The reference
project's last-write-wins rule means downstream-ordered modules override
upstream ones for the same variable. Reorder operations in the UI must
preserve this contract.

## `ModuleEntry` (discriminated by `type`)

```jsonc
{
  "id": "ab12cd34",
  "type": "fixed_values",
  "enabled": true,
  "collapsed": false,
  "meta": {
    "name": "style",
    "description": "",
    "tags": []
  },
  "entries": [
    { "variable_name": "subject", "value": "a cat" },
    { "variable_name": "style",   "value": "watercolor" }
  ]
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | `string` | yes | 8-char hex. Stable per-module across saves. Generated by `newModuleId()` (`crypto.getRandomValues`) — never reused. |
| `type` | `"fixed_values"` (extensible) | yes | Discriminator. New module types add new literals to this union; readers MUST treat unknown types as no-ops. |
| `enabled` | `boolean` | yes | When `false`, the runtime skips the module entirely. Frontend conflict scans also skip disabled modules. |
| `collapsed` | `boolean` | no (UI-only) | Persists the card's collapsed state across reloads. Engine ignores it. |
| `meta` | `ModuleMeta` | yes | Display metadata (name shown in card header, description for tooltips, tags for SPA library filtering). |
| `entries` | depends on `type` | yes | Type-specific payload. For `fixed_values`: `Array<{variable_name, value}>`. |

### `ModuleMeta`

| Field | Type | Default | Notes |
|---|---|---|---|
| `name` | `string` | `""` | Human-friendly label for the card. Empty renders as `"(unnamed)"`. |
| `description` | `string` | `""` | Optional long-form text. Reserved for SPA library tooltips. |
| `category` | `string` | `""` | SPA library categorization. Frontend currently doesn't render this. |
| `tags` | `string[]` | `[]` | SPA library filtering. Frontend currently doesn't render this. |

### `fixed_values` entries

```jsonc
{ "variable_name": "subject", "value": "a cat" }
```

| Field | Type | Notes |
|---|---|---|
| `variable_name` | `string` | Stored **bare** — no leading `$`. The sigil is rendering chrome only, applied by the UI when displaying the chip / preview. Engine reads bare names too. |
| `value` | `string` | Literal string the variable resolves to. Empty strings are valid (drop a variable to "nothing"). |

## Variable name normalization

Both sides MUST normalize variable names identically:

```python
# engine + frontend
name = raw_name.replace("$", "", 1).strip()
```

- Strip a single leading `$` (defensive — historical workflows that stored
  the sigil should still load).
- Strip surrounding whitespace.
- Empty after normalization → variable is dropped from the runtime context.

The frontend regex used for entry validation:
```ts
/^[A-Za-z_][A-Za-z0-9_]*$/
```

Names that don't match this regex still SAVE (we don't want to lose user
edits mid-typing) but render with an invalid badge in the editor. The
engine drops invalid names silently from the runtime context.

## Recovery & versioning

When the widget loads malformed JSON:

1. `parseWidgetJsonWithRecovery` returns `{value: emptyContextValue(), error, raw}`.
2. The Context node renders a recovery panel (red border, "View raw" / "Reset
   to empty" buttons) instead of the normal module list.
3. The user can copy the raw payload before resetting.
4. Reset clears the value and pushes an info toast confirming the action.

Engine-side equivalent: `module_from_dict` raises `ValueError` for missing
`id`/`type` or unknown `type`. The runtime should catch and skip rather
than abort the whole graph evaluation; an unrelated downstream node
shouldn't fail because one Context has corrupt data.

## Adding a new module type

1. **Engine:** add a dataclass in `engine/modules.py`, extend the
   `Module` union, add a branch in `module_from_dict`.
2. **Frontend types:** extend `ModuleEntry["type"]` literal in
   `src/widgets/_shared.ts`.
3. **Frontend UI:** extend the `kinds` array in
   `ModulePickerModal.vue`, add an icon + color, add an editor pane in
   `ModuleEditModal.vue`.
4. **Theme:** pick a hue from the brand palette in `theme.css` and add a
   `.type-<new_type>` rule for the card's type icon.
5. **Round-trip test:** Python `pytest` round-trips a fixture; Vitest
   verifies parseWidgetJson + serializeWidgetJson preserves shape.
6. **This doc:** add the new type's entry shape under the "ModuleEntry"
   section.
